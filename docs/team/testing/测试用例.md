## 2. 测试用例规范

### 2.1 用例编号规范

```
TC-[模块缩写]-[序号]
例如: TC-USER-001 (用户模块第一个用例)
```

### 2.2 优先级定义

| 优先级 | 说明 | 示例 |
|--------|------|------|
| P0 | 核心功能，阻塞发布 | 用户登录、支付流程 |
| P1 | 重要功能，影响用户体验 | 产品搜索、购物车 |
| P2 | 一般功能 | 个人信息修改 |
| P3 | 边缘功能 | 收藏功能 |

### 2.3 测试状态定义

| 状态 | 说明 |
|------|------|
| 待执行 | 尚未执行的测试用例 |
| 通过 | 测试通过 |
| 失败 | 测试失败，存在缺陷 |
| 阻塞 | 因环境或依赖问题无法执行 |

### 2.4 缺陷等级定义

| 等级 | 说明 | 示例 |
|------|------|------|
| 严重 | 系统崩溃、数据丢失 | 支付失败但扣款 |
| 主要 | 核心功能无法使用 | 无法登录、无法下单 |
| 次要 | 功能异常但有替代方案 | 搜索结果不准确 |
| 提示 | UI 问题、文案错误 | 错别字、样式偏差 |
