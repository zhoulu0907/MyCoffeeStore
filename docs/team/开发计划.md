## 📋 团队角色与职责

| 角色 | 职责 | 工作流 |
|------|------|--------|
| **架构师** | 设计前后端接口、数据结构、数据库设计 | 计划 → 设计 → 审查 |
| **前端开发** | React 页面开发、组件实现、API 对接 | 计划 → 开发 → 自测 |
| **后端开发** | Spring Boot 开发、API 实现、数据库操作 | 计划 → 开发 → 自测 |
| **测试工程师** | 测试用例设计、功能测试、缺陷跟踪 | 计划 → 测试 → 报告 |
| **QA 工程师** | 代码审查、质量检查、最佳实践验证 | 审查 → 反馈 → 验证 |

## 🔄 工作流规范

### 1. Worktree 策略
```
main (主分支)
├── architect      (架构师工作分支)
├── frontend       (前端工作分支)
├── backend        (后端工作分支)
├── testing        (测试工作分支)
└── qa             (QA 审查分支)
```

### 2. Git 提交规范
- 每个 worktree 完成后创建 PR
- Code Review 通过后合并到 main
- 本地 commit 记录进度
- **不推送到远程** (除非明确指示)

### 3. 文档规范
- 所有计划文档存放在 `docs/team/`
- 每个角色的工作记录在对应子目录
- 设计决策需有文档说明

## 📅 开发阶段规划

### 阶段一：架构设计 (架构师)
**Worktree**: `architect`

**输出**:
- `docs/team/architect/API接口设计.md`
- `docs/team/architect/数据库设计.md`
- `docs/team/architect/数据结构定义.md`

**任务清单**:
- [ ] RESTful API 接口定义 (仅 GET/POST)
- [ ] 数据库表结构设计
- [ ] VO/DO/DTO 类定义
- [ ] 前后端交互协议

---

### 阶段二：项目初始化 (前后端并行)
**Worktree**: `frontend` / `backend`

**前端输出**:
- React + Vite + TypeScript 项目结构
- 路由配置
- 状态管理 (React Context)
- UI 组件库搭建

**后端输出**:
- Spring Boot 3.x + MyBatis-Flex 项目结构
- MySQL 数据库连接配置
- 基础配置类
- 统一响应格式

---

### 阶段三：核心功能开发

#### 3.1 用户模块
**前端**: 登录/注册页面
**后端**: 用户认证 API (JWT)
**测试**: 登录注册测试用例

#### 3.2 咖啡产品模块
**前端**: 咖啡列表/详情页面
**后端**: 咖啡 CRUD API
**测试**: 产品管理测试用例

#### 3.3 购物车模块
**前端**: 购物车页面
**后端**: 购物车 API
**测试**: 购物车功能测试用例

#### 3.4 订单模块
**前端**: 订单列表/详情页面
**后端**: 订单创建/查询 API
**测试**: 订单流程测试用例

---

### 阶段四：测试与 QA
**测试工程师**:
- 编写完整测试用例
- 执行功能测试
- 生成测试报告

**QA 工程师**:
- 代码质量审查
- 最佳实践检查
